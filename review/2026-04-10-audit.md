# Agentia Audit — 2026-04-10

## Scope
Deep audit of current working tree against current user-facing surface (host CLI, agent CLI, AgentServer API, session/conversation management, files API, cleanup command).

## Findings summary

### Strong areas
- Core agent/host/server split is coherent.
- Session management is real and mostly aligned with specs.
- Recent pi adapter fix improves surfacing of upstream model errors.
- Host cleanup direction is conservative and sensible.

### Major issues
1. Test story is weak and not aligned to current product surface.
2. Legacy relay/moderator-oriented tests remain while current CLI/API commands lack proper regression coverage.
3. Some user-facing commands have misleading success exit codes on not-found operations.
4. Specs/docs overstate implementation maturity in places.
5. Concrete correctness bugs exist in conversation/session bookkeeping.

## Concrete issues to fix
- `_smart_route()` writes `.active/<agent>.jsonl` with `session_name=conv_id` in some paths instead of actual session name.
- `SessionManager.delete_session()` deletes `self._sessions[name]` instead of resolved key.
- AgentServer files PUT handler computes created-vs-updated after write, so created detection is wrong.
- `conv rename/delete` return success on missing conversations; bad for automation.
- No current-product test suite for host CLI + AgentServer API.

## Recommended next steps
1. Fix correctness bugs first.
2. Add focused automated tests for current user-facing commands.
3. Demote or replace legacy relay-era tests.
4. Tighten docs/spec claims where needed.
